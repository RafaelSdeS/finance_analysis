# Anchored Rolling Windows

This is how the agent is trained — always. `python -m src.agent.trainer`
trains one PPO model per window and reports results across all of them;
there is no separate "rolling eval" command and no fixed-split alternative.

## Motivation

A single train/val/test split only tests whether one policy, trained once on
past data, generalizes to one future period. Real investors retrain
continuously as new market data arrives — anchored rolling windows simulate
that:

- Each window trains on progressively larger historical data (always anchored
  at the dataset's start)
- Tests on a non-overlapping future period
- No lookahead bias (train/test never overlap; each window's train span is
  additionally tail-carved into train/val for early stopping, never touching
  the test span)
- Produces 8+ independent backtest evaluations across different market
  regimes instead of one

## Architecture

```
Window 0: train [2000–2010] → test [2010–2012]
Window 1: train [2000–2012] → test [2012–2014]
Window 2: train [2000–2014] → test [2014–2016]
...
Window 7: train [2000–2024] → test [2024–2026]   ← most recent = production model
```

Generated by `config.generate_windows()` from `AgentConfig.dataset_start`/
`dataset_end`/`window_train_years`/`window_test_years`. Each window's train
span is further tail-carved by `window_val_fraction` (default 0.15) via
`config.window_to_config()`, which builds a self-consistent per-window
`AgentConfig` (train/val from the carve, test untouched).

**Key properties:**
- ✅ **No lookahead**: test data for window N is not in train for window N+1; the val carve-out never touches a window's own test span
- ✅ **Expanding history**: each model sees more years than the last (realistic for learning)
- ✅ **Multiple regimes**: windows span 2008 crash, COVID, rate hikes, etc.
- ✅ **Independent backtest**: each window is a separate evaluation
- ✅ **One production model**: the most recent window's model is trained as `agent_best.zip`/`agent_final.zip` inside the run's scratch dir, then promoted to the stable top-level `data/models/agent_{best,final}.zip` — the same filenames `evaluate.py`/`infer.py`/`run_allocation.py` already default to

## Usage

### Quick smoke test (many small windows, fast):
```bash
python -m src.agent.trainer --train-years 2 --test-years 1 --timesteps 2048
```

### Full training (8 windows, 1M timesteps/window — the default):
```bash
python -m src.agent.trainer
```
(Takes several hours on GPU — 8x a single 1M-timestep run, since every window trains independently.) Each invocation gets its own `data/models/runs/<session_id>/` + `data/logs/agent/runs/<session_id>/` — a fresh run never collides with a previous one, no manual cleanup needed.

### Resume an interrupted run:
```bash
python -m src.agent.trainer --resume
```
Resumes the most-recently-modified `runs/<session_id>/` directory: whichever
window in it is currently incomplete (no `_final.zip` yet) picks up from its
latest checkpoint; already-finished windows are skipped automatically.

## Output

- `data/models/runs/<session_id>/{window_{id},agent}_{best,final}.zip` — all windows' models for this run (checkpoints removed once each window finishes)
- `data/models/agent_{best,final}.zip` — the most recent window's model, promoted to the stable production path
- `data/models/rolling_eval_results.json` — per-window metrics + aggregated summary (mean/std/min/max per strategy)
- `data/backtest/walkforward_results.parquet` + `walkforward_metrics.json` — all windows' out-of-sample rollouts stitched into one continuous curve

Example summary structure (`rolling_eval_results.json`):
```json
{
  "summary": {
    "agent": {
      "sharpe": {"mean": 0.72, "std": 0.15, "min": 0.48, "max": 0.91},
      "max_drawdown": {"mean": 0.25, "std": 0.08, "min": 0.15, "max": 0.35}
    },
    "equal_weight": { ... }
  },
  "windows": [ ... ]
}
```

## Interpretation

**What matters:**
- **Agent vs equal_weight Sharpe**: if mean is +0.2+ and consistent (low std), agent has learned something robust.
- **Drawdown std**: if high, performance is regime-dependent (not robust).
- **Min/max spread**: wide spread = high variance (lucky vs unlucky windows).

**Example reads:**
- ✅ Agent mean Sharpe 0.75±0.08, equal-weight 0.70±0.10 → agent outperforms reliably
- ⚠️ Agent mean Sharpe 0.80±0.25 → volatile (good in some regimes, bad in others)
- ❌ Agent mean Sharpe 0.65±0.15, equal-weight 0.70±0.10 → agent underperforms

## Implementation

- `config.py`: `generate_windows()`, `window_to_config()`, `RollingWindow`
- `trainer.py`: `train()` — PPO + val-Sharpe early stopping + checkpoints for one window; `main()` — the sole training CLI
- `rolling_eval.py`: `run_rolling_eval()` — loops windows, delegates each to `trainer.train_window()`/`train()`; `eval_window()` — backtests all strategies on a window's test split; `finalize_and_report()` — stitches + summarizes + persists results

## Future Enhancements

- **Parallel training**: spawn N windows in parallel (each on separate GPU if available)
- **Per-window scaler**: currently one global `StandardScaler` (fit on the earliest window's train span, conservative/leak-free) is reused for every window; a per-window scaler would be more precise but adds real complexity
- **Hyperparameter tuning**: per-window or global meta-validation
